# AI 工程术语表

## A

### Agent
- **大家怎么说：** "一个能自主思考和行动的 AI"
- **实际含义：** 一个 while 循环——LLM 决定下一步调用哪个工具、执行它、查看结果，然后重复这个过程
- **名称由来：** 借用自哲学概念——"agent"是指任何能在世界中采取行动的事物。在 AI 领域，它其实就是"LLM + 工具 + 循环"

### Attention
- **大家怎么说：** "AI 如何关注重点内容"
- **实际含义：** 一种机制，每个 token 会计算所有其他 token 值的加权求和，权重由它们之间的相关性决定（通过 query 和 key 向量的点积计算）
- **名称由来：** 2017 年的论文《Attention Is All You Need》用人类的选择性注意力来类比命名

### Alignment
- **大家怎么说：** "让 AI 变安全"
- **实际含义：** 让 AI 系统的行为与人类意图、价值观和偏好保持一致的技术挑战，包括设计者没有预料到的边缘情况

### Autoregressive
- **大家怎么说：** "AI 一次生成一个词"
- **实际含义：** 一种模型，基于前面所有 token 来预测下一个 token，然后将预测结果作为下一步的输入。GPT、LLaMA 和 Claude 都是自回归模型。

### Activation Function
- **大家怎么说：** "层与层之间的那个非线性函数"
- **实际含义：** 在每个线性层之后施加的函数，用来引入非线性。没有它，无论堆叠多少层线性层，最终都会坍缩成一个线性变换。ReLU、GELU 和 SiLU 是最常见的激活函数。选择哪种直接影响训练过程中梯度能否正常流动。

### Adam (Optimizer)
- **大家怎么说：** "默认的优化器"
- **实际含义：** Adaptive Moment Estimation（自适应矩估计）。结合了动量（一阶矩）和每个参数的自适应学习率（二阶矩）。对早期步数有偏差校正。在大多数任务上无需太多调参就能表现良好。

### AdamW
- **大家怎么说：** "更好的 Adam"
- **实际含义：** 带有解耦权重衰减的 Adam。在标准 Adam 中，L2 正则化会被每个参数的自适应学习率缩放，这不是你想要的效果。AdamW 直接对权重施加衰减，与梯度统计无关。是训练 transformer 的默认优化器。

### Autograd
- **大家怎么说：** "自动求梯度"
- **实际含义：** 一个记录 tensor 上操作并通过反向模式自动微分来计算梯度的系统。PyTorch 的 autograd 动态构建计算图（动态图），而 JAX 使用函数变换（grad）。正是它让反向传播变得可行——你只写前向传播，框架负责计算所有导数。

## B

### Batch Size
- **大家怎么说：** "一次处理多少个样本"
- **实际含义：** 在一次前向/反向传播中处理的训练样本数量，之后才更新权重。更大的 batch 给出更稳定的梯度估计，但占用更多内存。典型值：训练时 32-512，推理时可以更大。Batch size 与学习率相互影响——batch 翻倍，学习率也翻倍（线性缩放法则）。

### Backpropagation
- **大家怎么说：** "神经网络怎么学习的"
- **实际含义：** 一种算法，通过在网络中反向应用链式法则来计算每个权重对误差的贡献程度，然后按比例调整权重
- **名称由来：** 误差从输出层逐层向输入层反向传播

## C

### Context Window
- **大家怎么说：** "AI 能记住多少东西"
- **实际含义：** 单次 API 调用中能容纳的最大 token 数（输入 + 输出）。不是记忆——它是一个固定大小的缓冲区，每次调用都会重置

### Chain of Thought (CoT)
- **大家怎么说：** "让 AI 一步步思考"
- **实际含义：** 一种提示技术，要求模型展示推理步骤。这能提升多步骤问题的准确率，因为每个步骤都会为下一个 token 的生成提供上下文

### CNN (Convolutional Neural Network)
- **大家怎么说：** "图像 AI"
- **实际含义：** 一种使用卷积运算（在输入上滑动滤波器）来检测局部模式的神经网络。堆叠卷积层可以检测越来越复杂的特征：边缘、纹理、物体。

### CUDA
- **大家怎么说：** "GPU 编程"
- **实际含义：** NVIDIA 的并行计算平台。可以让你在数千个 GPU 核心上同时运行矩阵运算。PyTorch 和 TensorFlow 底层都使用 CUDA。

### Chunking
- **大家怎么说：** "把文档切分成小块"
- **实际含义：** 在 embedding 用于检索之前，将文本切分成段。Chunk 大小决定了搜索结果的粒度。太小：丢失上下文。太大：稀释相关性。常见策略：固定大小加重叠、基于句子、或语义切分。典型 chunk 大小：256-512 token，重叠 10-20%。

### Contrastive Learning
- **大家怎么说：** "通过对比来学习"
- **实际含义：** 在 embedding 空间中，拉近相似样本对、推远不相似样本对的训练方式。CLIP 就用这种方式：匹配的图文对 vs 不匹配的图文对。

### Cosine Similarity
- **大家怎么说：** "两个向量有多相似"
- **实际含义：** 两个向量之间夹角的余弦值：dot(a, b) / (||a|| * ||b||)。范围从 -1（方向相反）到 1（方向相同）。忽略大小，只关注方向。是 embedding 和语义搜索的标准相似度度量。

### Cross-Entropy
- **大家怎么说：** "分类损失函数"
- **实际含义：** 衡量两个概率分布之间的差异。分类任务中：-sum(y_true * log(y_pred))。语言模型中：正确下一个 token 的负对数概率。越低越好。Perplexity 就是 cross-entropy 的指数形式。

## D

### Data Augmentation
- **大家怎么说：** "造更多训练数据"
- **实际含义：** 对已有数据创建修改后的副本（旋转图片、加噪声、改写文本），在不收集新数据的情况下增加训练集多样性。减少过拟合。

### Decoder
- **大家怎么说：** "输出部分"
- **实际含义：** 在 transformer 中，decoder 使用因果（带掩码的）自注意力，使每个位置只能关注前面的位置。GPT 是纯 decoder。BERT 是纯 encoder。T5 是 encoder-decoder。

### Diffusion Model
- **大家怎么说：** "从噪声生成图像的 AI"
- **实际含义：** 一种训练来逆转逐步加噪过程的模型——它学习预测和去除噪声，生成时从纯噪声开始迭代去噪

### DPO (Direct Preference Optimization)
- **大家怎么说：** "更简单的 RLHF"
- **实际含义：** 一种完全跳过 reward model 的训练方法——直接优化语言模型，使其在人类偏好的成对比较中倾向于更好的回答

### Dropout
- **大家怎么说：** "随机关闭神经元"
- **实际含义：** 训练时随机将一部分激活值设为零。迫使网络不依赖任何单个神经元。推理时关闭。简单但有效的正则化手段。

## E

### Eigenvalue
- **大家怎么说：** "PCA 用的那个数学玩意"
- **实际含义：** 对于矩阵 A，特征值 lambda 满足 Av = lambda*v（其中 v 是某个向量）。它告诉你矩阵在该方向上对向量的缩放程度。大特征值 = 数据中高方差的方向。

### Embedding
- **大家怎么说：** "把词变成数字的某种 AI 魔法"
- **实际含义：** 一种学习到的映射，将离散项（词、图像、用户）映射到连续空间中的稠密向量，相似的项在空间中彼此靠近
- **名称由来：** 这些项被"嵌入"（embedded）到一个距离有意义的几何空间中

### Encoder
- **大家怎么说：** "输入部分"
- **实际含义：** 在 transformer 中，encoder 使用双向自注意力，使每个位置可以关注所有位置。BERT 是纯 encoder。适合理解类任务（分类、命名实体识别），但不适合生成。

### Epoch
- **大家怎么说：** "完整过一遍数据"
- **实际含义：** 就是字面意思。完整遍历训练集中的每个样本一次。多个 epoch = 多次看到同样的数据。更多 epoch 可以改善学习效果，但有过拟合风险。

## F

### Feature
- **大家怎么说：** "数据中的一列"
- **实际含义：** 数据的一个可度量的属性。在传统机器学习中，你手动设计特征。在深度学习中，网络从原始数据中自动学习特征。

### Few-Shot
- **大家怎么说：** "先给 AI 几个示例"
- **实际含义：** 在 prompt 中包含少量输入-输出示例，然后再让模型执行任务。通常 3-5 个示例。模型通过这些示例进行模式匹配，理解期望的格式和行为。与 zero-shot（无示例）和微调（数千个示例融入权重）形成对比。

### Fine-tuning
- **大家怎么说：** "用你的数据训练 AI"
- **实际含义：** 从预训练模型的权重出发，在更小的、特定任务的数据集上继续训练。只更新已有权重，不会从零学习新知识

### Function Calling
- **大家怎么说：** "能使用工具的 AI"
- **实际含义：** LLM 请求执行外部函数的结构化方式。你用 JSON Schema 描述定义工具，模型输出一个指定调用哪个函数及参数的 JSON 对象，你的代码执行它，结果返回给模型。和 agent 不一样——function calling 是机制，agent 是循环。

## G

### Guardrails
- **大家怎么说：** "AI 的安全过滤器"
- **实际含义：** 围绕 LLM 的输入/输出验证层，用于检测和阻止有害内容、prompt 注入攻击、PII 泄露或离题回答。通常是一个流水线：输入过滤 -> LLM -> 输出过滤。可以基于规则（正则表达式、关键词列表）或基于模型（对安全性打分的分类器）。

### GPT
- **大家怎么说：** "ChatGPT" 或 "那个 AI"
- **实际含义：** Generative Pre-trained Transformer——一种特定架构，使用纯 decoder 的 transformer 在大规模文本语料上训练，预测下一个 token
- **名称由来：** Generative（生成文本）、Pre-trained（先在大规模数据上训练，再适配）、Transformer（架构名）

### GAN (Generative Adversarial Network)
- **大家怎么说：** "两个 AI 互相对抗"
- **实际含义：** 一个生成器网络试图创建逼真的数据，同时一个判别器网络试图区分真假。它们一起训练：生成器越来越擅长骗过判别器，判别器越来越擅长识别伪造品。

### Gradient
- **大家怎么说：** "斜率"
- **实际含义：** 一个偏导数向量，指向最陡上升方向。在机器学习中，沿着梯度的反方向（梯度下降）来最小化损失。

### Gradient Descent
- **大家怎么说：** "AI 如何改进"
- **实际含义：** 一种优化算法，沿着损失函数下降最陡的方向调整参数，就像在高维地形中下山

## H

### Hyperparameter
- **大家怎么说：** "你要调的那些设置"
- **实际含义：** 训练前设定的、控制训练过程本身的值：学习率、batch size、层数、dropout 比例。与模型参数（权重）不同，这些不是从数据中学到的。

### Hallucination
- **大家怎么说：** "AI 在撒谎" 或 "瞎编"
- **实际含义：** 模型生成了听起来合理但没有依据的文本——它在做模式补全，而不是事实检索

## I

### Inference
- **大家怎么说：** "跑 AI"
- **实际含义：** 用训练好的模型对新数据做预测。不更新权重。这就是你在生产环境中做的事：发送输入，得到输出。

### Inductive Bias
- **大家怎么说：** 从没听说过
- **实际含义：** 模型架构中内置的假设。CNN 假设局部模式重要（卷积）。RNN 假设顺序重要（序列处理）。Transformer 假设所有位置可能相互关联（attention）。正确的归纳偏置帮助模型用更少的数据更快学习。

### JAX
- **大家怎么说：** "Google 的机器学习框架"
- **实际含义：** 一个兼容 NumPy 的库，增加了自动微分（grad）、JIT 编译（jit）、自动向量化（vmap）和多设备并行（pmap）。与 PyTorch 的面向对象风格不同，JAX 是纯函数式的——没有隐藏状态，没有原地修改。被 Google DeepMind 用于 AlphaFold、Gemini 和大规模研究。

## K

### KV Cache
- **大家怎么说：** "让推理更快"
- **实际含义：** 在自回归生成过程中，缓存之前 token 的 key 和 value 矩阵，这样每一步就不需要重新计算。用内存换速度。快速 LLM 推理的关键。

## L

### Latent Space
- **大家怎么说：** "隐藏的表示空间"
- **实际含义：** 一种压缩的、学习到的表示空间，相似的输入映射到相近的点。Autoencoder、VAE 和 diffusion 模型都在 latent space 中工作。维度低于输入，但保留了重要结构。

### Learning Rate
- **大家怎么说：** "AI 学得多快"
- **实际含义：** 控制梯度下降步长的标量。太高：越过最小值发散。太低：收敛太慢或卡住。最重要的超参数。

### LLM (Large Language Model)
- **大家怎么说：** "AI" 或 "大脑"
- **实际含义：** 基于 transformer 的神经网络，训练来预测序列中的下一个 token，拥有数十亿参数，在互联网规模的文本数据上训练

### LoRA (Low-Rank Adaptation)
- **大家怎么说：** "高效的微调方法"
- **实际含义：** 不更新所有权重，而是在原始权重旁边插入小型低秩矩阵。只训练这些小矩阵，内存减少 10-100 倍

### Loss Function
- **大家怎么说：** "AI 错了多少"
- **实际含义：** 衡量预测输出与实际输出之间差距的函数。训练就是最小化这个函数。回归用 MSE，分类用 cross-entropy，embedding 用对比损失。损失函数的选择定义了模型眼中"好"的含义。

## M

### Mixed Precision
- **大家怎么说：** "加速训练的技巧"
- **实际含义：** 前向传播和大部分运算使用 float16（更快、更省内存），但梯度累积和权重更新保持 float32（更精确）。可获得 2 倍加速，精度损失可忽略不计。

### MoE (Mixture of Experts)
- **大家怎么说：** "只有部分模型在运行"
- **实际含义：** 一种包含多个"专家"子网络的模型，路由机制将每个输入只发送给少数几个专家。完整模型很大，但每次前向传播很便宜，因为大部分专家被跳过了。Mixtral 和 GPT-4 使用这种架构。

### MCP (Model Context Protocol)
- **大家怎么说：** "让 AI 使用工具的一种方式"
- **实际含义：** 一个开放协议（基于 stdio/HTTP 的 JSON-RPC），标准化了 AI 应用连接外部数据源和工具的方式，为工具、资源和 prompt 提供类型化 schema

## N

### NaN (Not a Number)
- **大家怎么说：** "训练崩了"
- **实际含义：** 一个表示未定义结果（0/0、inf-inf）的浮点值。训练中出现 NaN loss 通常意味着：学习率太高、梯度爆炸、对零取对数、或除以零。训练失败时永远是第一个要排查的。

### Normalization
- **大家怎么说：** "缩放数据"
- **实际含义：** 将值调整到标准范围。Batch normalization 在 batch 维度上归一化。Layer normalization 在特征维度上归一化。两者都能稳定训练并允许使用更高的学习率。

## O

### Overfitting
- **大家怎么说：** "模型把数据背下来了"
- **实际含义：** 模型在训练数据上表现好，但在未见数据上表现差。它学到了噪声，而不是信号。解决方法：更多数据、正则化（dropout、权重衰减）、早停、数据增强、简化模型。

### Optimizer
- **大家怎么说：** "更新权重的那个东西"
- **实际含义：** 用梯度更新模型参数的算法。SGD 最简单。Adam 最常用。每个优化器有不同的特性：收敛速度、内存占用、对超参数的敏感度。

## P

### Parameter
- **大家怎么说：** "模型大小"
- **实际含义：** 模型中一个可学习的值，通常是权重或偏置。"7B 参数"意味着 70 亿个可学习的数字。每个 float32 参数占 4 字节，所以 7B 参数 = 仅权重就需要 28GB 内存。

### Perplexity
- **大家怎么说：** "模型有多困惑"
- **实际含义：** 平均 cross-entropy 损失的指数形式。越低越好。Perplexity 为 10 意味着模型的不确定性相当于在每一步从 10 个 token 中均匀选择。

### Precision & Recall
- **大家怎么说：** "准确率指标"
- **实际含义：** Precision = 你标记的项目中，有多少是正确的。Recall = 所有正确的项目中，你找到了多少。两者此消彼长：抓住所有垃圾邮件（高 recall）意味着更多误报（低 precision）。F1 分数是它们的调和平均。误报代价高时用 precision，漏报代价高时用 recall。

### Prompt Engineering
- **大家怎么说：** "用正确的方式跟 AI 对话"
- **实际含义：** 设计输入文本以可靠地产出期望输出——包括 system prompt、few-shot 示例、格式指令和 chain-of-thought 触发器

### Prompt Injection
- **大家怎么说：** "用文字黑掉 AI"
- **实际含义：** 一种攻击方式，输入中的恶意文本覆盖了 system prompt 或指令。直接注入：用户输入"忽略之前的指令"。间接注入：检索到的文档中包含隐藏指令。相当于 LLM 领域的 SQL 注入。目前没有完美解决方案——防御手段是输入验证、输出过滤和权限分离的多层组合。

## Q

### QLoRA
- **大家怎么说：** "更便宜的 LoRA"
- **实际含义：** 量化版 LoRA。将冻结的基础模型权重保持在 4 位精度（NF4 格式），同时以 16 位训练 LoRA adapter。比标准 LoRA 再减少 3-4 倍内存。一个用 LoRA 需要 14GB 的 7B 模型，用 QLoRA 只需 4-6GB。在大多数基准测试上质量与完整微调相差不到 1%。

## R

### RAG (Retrieval-Augmented Generation)
- **大家怎么说：** "能搜索的 AI"
- **实际含义：** 一种模式——从知识库中检索相关文档（使用 embedding 相似度），塞进 prompt 中，然后让 LLM 基于该上下文回答问题
- **名称由来：** Retrieval（检索文档）+ Augmented（加入 prompt）+ Generation（LLM 生成答案）

### RLHF (Reinforcement Learning from Human Feedback)
- **大家怎么说：** "他们怎么让 AI 变得有用的"
- **实际含义：** 一个训练流水线：(1) 收集人类对模型输出的偏好，(2) 基于这些偏好训练 reward model，(3) 用 PPO 优化 LLM 使其产生更高奖励的输出

### Quantization
- **大家怎么说：** "让模型变小"
- **实际含义：** 将模型权重的精度从 float32（4 字节）降到 int8（1 字节）或 int4（0.5 字节）。用少量精度换取 4-8 倍的内存节省和更快的推理。GPTQ、AWQ 和 GGUF 是常见格式。

### ReLU
- **大家怎么说：** "激活函数"
- **实际含义：** Rectified Linear Unit：f(x) = max(0, x)。最简单的非线性激活函数。计算快，正值不会饱和。到处都在用，因为它好用且廉价。变体：LeakyReLU、GELU、SiLU。

### ROUGE
- **大家怎么说：** "摘要评估指标"
- **实际含义：** Recall-Oriented Understudy for Gisting Evaluation。衡量生成文本与参考文本的重叠程度。ROUGE-1 统计单词匹配，ROUGE-2 统计二元组匹配，ROUGE-L 找最长公共子序列。计算成本低，但只衡量表面相似度——意思相同但措辞不同的句子得分会很低。

## S

### Semantic Search
- **大家怎么说：** "理解语义的智能搜索"
- **实际含义：** 按语义而非关键词匹配来查找文档。将查询和所有文档 embedding 到同一个向量空间，然后返回与查询 embedding 最近的文档。"支付失败"能找到"交易被拒"，即使它们没有共同的词。由 embedding 模型 + 向量数据库驱动。

### Streaming
- **大家怎么说：** "看着回答一个字一个字地出现"
- **实际含义：** LLM 在生成 token 的同时就发送，而不是等完整响应生成完毕。使用 Server-Sent Events (SSE) 或 WebSocket 协议。将首个 token 的感知延迟从秒级降到毫秒级。生产级聊天界面的必备功能。每个 chunk 包含一个 delta（部分 token 或词）。

### Self-Attention
- **大家怎么说：** "模型如何决定关注什么"
- **实际含义：** 每个 token 计算 query、key 和 value 向量。两个 token 之间的 attention 权重 = 它们 query 和 key 的点积，经过缩放和 softmax。输出 = value 向量的加权求和。让每个 token 都能看到其他所有 token。

### SFT (Supervised Fine-Tuning)
- **大家怎么说：** "教模型遵循指令"
- **实际含义：** 在（指令，回答）对上微调预训练模型。模型学习在给定指令的情况下生成回答。这就是把 base model 变成 chat model 的过程。

### Softmax
- **大家怎么说：** "把数字变成概率"
- **实际含义：** softmax(x_i) = exp(x_i) / sum(exp(x_j))。将任意实数向量转换为概率分布（全部为正，总和为 1）。用于分类头、attention 权重，以及任何需要概率的地方。

### Swarm
- **大家怎么说：** "一群像蜜蜂一样协作的 AI agent"
- **实际含义：** 多个 agent 共享状态并通过消息传递协调，涌现行为来自简单的个体规则而非中央控制

## T

### System Prompt
- **大家怎么说：** "AI 的指令"
- **实际含义：** 对话开头的特殊消息，设定模型的行为、人设和约束。在用户消息之前处理。在大多数界面中对用户不可见。定义了模型应该做什么和不应该做什么、语气、格式偏好和领域重点。与 user prompt 不同——system prompt 由开发者设定。

### Tensor
- **大家怎么说：** "多维数组"
- **实际含义：** 深度学习框架中的基础数据结构。0 维 tensor 是标量，1 维是向量，2 维是矩阵，3 维及以上是 tensor。在 PyTorch 和 JAX 中，tensor 会追踪其计算历史以支持自动微分，可以存放在 CPU 或 GPU 上。所有神经网络的输入、输出、权重和梯度都是 tensor。

### Token
- **大家怎么说：** "一个词"
- **实际含义：** 由 BPE 等 tokenizer 产生的子词单元（英文中通常 3-4 个字符）。"unbelievable" 可能是 3 个 token："un" + "believ" + "able"

### Temperature
- **大家怎么说：** "创意调节"
- **实际含义：** 在 softmax 之前除以 logits 的标量。Temperature=1 是默认值。越高 = 分布越平坦 = 输出越随机。越低 = 分布越尖锐 = 输出越确定。Temperature=0 等于 argmax（总是选概率最高的 token）。

### Transfer Learning
- **大家怎么说：** "用预训练模型"
- **实际含义：** 将在一个任务上训练的模型适配到另一个任务。早期层学到的通用特征（边缘、语法模式）可以迁移。只有后面几层需要特定任务的训练。这就是为什么你可以把 BERT 微调用于任何 NLP 任务。

### Transformer
- **大家怎么说：** "现代 AI 背后的架构"
- **实际含义：** 一种神经网络架构，使用 self-attention（让每个位置都能关注其他所有位置）而非循环来处理序列，从而实现大规模并行化
- **名称由来：** 通过 attention 层将输入表示变换（transform）为输出表示

## U

### Underfitting
- **大家怎么说：** "模型没学到东西"
- **实际含义：** 模型太简单，无法捕捉数据中的模式。训练损失一直很高。解决方法：更多参数、更多层、更长训练、降低正则化、更好的特征。

## V

### VAE (Variational Autoencoder)
- **大家怎么说：** "一种生成模型"
- **实际含义：** 一种通过强制编码器输出服从高斯分布来学习平滑 latent space 的 autoencoder。你可以从这个分布中采样并解码来生成新数据。重参数化技巧使其可以通过反向传播训练。

### Vector Database
- **大家怎么说：** "AI 专用的数据库"
- **实际含义：** 一种针对存储向量（浮点数稠密数组）和快速近似最近邻搜索优化的数据库。是相似度搜索、RAG 和推荐系统的核心组件。

## W

### Weight
- **大家怎么说：** "模型学到的东西"
- **实际含义：** 模型参数矩阵中的一个数字。一个输入维度 768、输出维度 3072 的线性层有 768*3072 = 2,359,296 个权重。训练就是调整每个权重以最小化损失函数。

### Weight Decay
- **大家怎么说：** "正则化"
- **实际含义：** 在损失函数中加入与权重大小成比例的惩罚项。等价于 L2 正则化。防止权重变得过大。典型值：0.01-0.1。

## Z

### Zero-Shot
- **大家怎么说：** "不需要训练"
- **实际含义：** 将模型用于它没有被明确训练过的任务，且 prompt 中不包含该任务的示例。模型从预训练中泛化。之所以可行，是因为大模型见过足够多的多样性来处理新的任务格式。
