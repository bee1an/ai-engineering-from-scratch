# LLM 的差分隐私

> DP-SGD 仍然是标准方法 — 注入噪声的梯度更新提供形式化的 (epsilon, delta) 保证。计算、内存和效用方面的开销很大；参数高效的 DP 微调（LoRA + DP-SGD）是 2025 年的常见配置（ACM 2025）。两组证据存在张力：基于金丝雀的成员推断（Duan et al., 2024）报告对语言模型的成功率有限；训练数据提取（Carlini et al., 2021; Nasr et al., 2025）恢复了大量逐字记忆。解决方案（arXiv:2503.06808, 2025 年 3 月）：差距在于测量的内容不同 — 插入的金丝雀 vs "最可提取"数据。新的金丝雀设计实现了无需影子模型的基于损失的 MIA，并产生了首个在真实数据和现实 DP 保证下对 LLM 的非平凡 DP 审计。替代方案：PMixED (arXiv:2403.15638) — 通过下一 token 分布的专家混合在推理时实现私有预测；DP 合成数据生成（Google Research 2024）。新兴攻击：通过 LLM 反馈的差分隐私逆转 — 置信度分数泄露。

## 问题

LLM 会记忆。Carlini et al. 2021 表明生产语言模型可以按需逐字复现训练文本。差分隐私是形式化的防御：训练使得输出对任何单个训练样本可证明不敏感。2024-2025 年的证据表明 DP-SGD 是必要的，但部署的 ε 值可能与威胁模型不匹配。

## 概念

### (ε, δ)-差分隐私

随机算法 M 是 (ε, δ)-DP 的，如果对于任何仅差一个样本的两个数据集和任何事件 S：
P(M(D) in S) <= e^ε * P(M(D') in S) + δ。

解释：输出分布足够接近（由 ε 参数化），使得任何单个个体的贡献不能被可靠推断，除了概率 δ。

### DP-SGD

Abadi et al. 2016。标准步骤：
1. 采样一个 mini-batch。
2. 计算每个样本的梯度。
3. 将每个样本梯度裁剪到阈值 C。
4. 对裁剪后的梯度求和并添加标准差为 σ * C 的高斯噪声。
5. 使用带噪声的和来更新参数。

隐私代价由 accountant 追踪（Moments Accountant、Rényi DP accountant）。LLM 文献中报告的 ε 值因威胁模型、数据敏感性和效用目标而差异很大；没有普遍"安全"的默认 ε。已发表的例子在某些 LLM 训练设置中大致跨越 ε ≈ 1–10，但这些是说明性的 — 不是推荐默认值。更低的 ε 通常需要更多噪声，可能增加效用损失。

### LoRA + DP-SGD

对前沿模型进行完整 DP-SGD 是不可行的。LoRA（Hu et al. 2022）将梯度更新限制在小型适配器上，减少每样本梯度存储。LoRA + DP-SGD 是 2025 年的常见配置。DP 保证适用于适配器；基础模型保持固定。

### 2024-2025 年的张力

两条证据线：

- **金丝雀 MIA（Duan et al. 2024）。** 在训练数据中插入唯一金丝雀，测量成员推断攻击者能否识别它们。报告对语言模型的成功率有限。暗示 MIA 很难。
- **训练数据提取（Carlini 2021, Nasr et al. 2025）。** 用前缀提示模型；测量它是否恢复训练中的逐字文本。报告大量记忆。暗示 MIA 在相关意义上很容易。

2025 年 3 月的解决方案（arXiv:2503.06808）：两者测量的是不同的东西。MIA 问的是"样本 e 是否在 D 中？"针对插入的金丝雀。提取问的是"我能从 D 中恢复什么？""最可提取"的样本才是隐私关注的重点；金丝雀低估了这一点，因为它们没有被优化为可提取的。

新的金丝雀设计。无需影子模型的基于损失的 MIA。首个在真实数据和现实 DP 保证下对 LLM 的非平凡 DP 审计。

### DP 训练的替代方案

- **PMixED (arXiv:2403.15638)。** 推理时的私有预测。下一 token 分布上的专家混合；每个专家看到训练数据的一个分片；聚合时添加噪声以实现 DP。完全避免 DP 训练。
- **DP 合成数据生成（Google Research 2024）。** 用 DP-SGD 进行 LoRA 微调，采样合成数据，在合成数据上训练下游分类器。

两者都以不同的威胁模型为代价，绕过了完整 DP 训练的效用成本。

### 通过 LLM 反馈的差分隐私逆转

2025 年新兴攻击。使用 DP 训练模型的置信度分数作为预言机来重新识别个体。即使输出不泄露，置信度分布也可能泄露。

防御：不暴露置信度，或在暴露前截断/量化它们。这是 (ε, δ)-DP 训练之外的额外要求。

## Use It

`code/main.py` 在玩具二分类数据集上模拟 DP-SGD。你可以扫描噪声乘数 σ 和裁剪范数 C，追踪 (ε, δ) 预算和准确率代价。一个"金丝雀攻击"插入唯一训练样本，测量 DP 前后对数损失测试能否检测到它。

## Ship It

本课产出 `outputs/skill-dp-audit.md`。给定语言模型部署上的 DP 声明，它审计：(ε, δ) 值、使用的 accountant、MIA 评估协议，以及是否评估了置信度暴露向量。

## 练习

1. 运行 `code/main.py`。扫描 σ ∈ {0.5, 1.0, 2.0} 并报告 (ε, δ)-准确率权衡。找出效用崩溃的点。

2. 实现金丝雀插入和对数损失测试。测量 σ = 1.0 时 DP-SGD 前后的检测率。

3. 阅读 Nasr et al. 2025 关于训练数据提取的论文。为什么提取成功率在中等 ε 下不会崩溃？这对 MIA 作为评估意味着什么？

4. 设计一个使用 PMixED (arXiv:2403.15638) 的部署，完全在推理时运行。PMixED 解决了 DP-SGD 不解决的什么威胁模型？

5. 草拟通过 LLM 反馈的 DP 逆转攻击。设计一个限制置信度分数泄露的对策并估计其部署成本。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|----------|----------|
| DP | "(ε, δ)-差分隐私" | 形式化隐私：相邻数据集变化下输出分布接近 |
| DP-SGD | "注入噪声的 SGD" | 梯度裁剪 + 高斯噪声添加；标准 DP 训练 |
| LoRA + DP-SGD | "高效私有微调" | 在低秩适配器上进行 DP-SGD；2025 年标准配置 |
| MIA | "成员推断" | 确定某样本是否在训练数据中的攻击 |
| 金丝雀 | "插入的水印样本" | 用于测量 DP 泄露的唯一训练样本 |
| PMixED | "私有推理混合" | 通过下一 token 分布的专家混合实现推理时 DP |
| DP 逆转 | "置信度泄露攻击" | 使用模型置信度作为预言机进行重新识别的攻击 |
