# 心智社会与多 Agent 辩论

> Minsky 1986 年的前提——智能是专家的社会——每十年被重新发现一次。2023 年 Du 等人将其变成了一个具体算法：多个 LLM 实例提出答案、阅读彼此的答案、批评并更新。经过 N 轮后，它们收敛到一个共识，在六个推理和事实性任务上击败了零样本 CoT 和反思。两个发现很重要：**多个 agent** 和**多轮**各自独立贡献。社会击败单 agent 独白；多轮交换击败一次性投票。

**Type:** Learn + Build
**Languages:** Python (stdlib)
**Prerequisites:** Phase 16 · 04 (Primitive Model)
**Time:** ~60 minutes

## 问题

自一致性——对一个模型采样多次并取多数答案——是你能加装的最便宜的推理改进。它有效，但很快饱和。你可以将样本翻倍却看不到另一个有意义的跳跃。

辩论打破了饱和。不是从一个模型取 N 个独立样本，而是 N 个 agent 阅读彼此的推理并修订。样本之间的相关性下降（它们不再是 i.i.d.），收敛点往往是正确的，而 i.i.d. 投票在那里是自信地错误的。

## 概念

### Du 等人 2023 算法

来自 arXiv:2305.14325（ICML 2024）：

1. N 个 agent 各自对问题产出一个初始答案。
2. 对于轮次 r = 2..R：每个 agent 被展示其他 agent 第 r-1 轮的答案，并被要求"考虑这些，给出你更新的答案。"
3. R 轮后，对最终答案进行多数投票。

论文在 MMLU、GSM8K、传记、MATH 和事实性基准上测试。辩论一致击败 CoT 和自我反思。

### 两个独立旋钮

来自同一论文的消融实验：

- **仅 agent 数量**（1 轮，N 的多数投票）在大多数任务上击败单 agent，但会饱和。
- **仅轮数**（1 个 agent 看自己先前的推理）几乎没有帮助——反思的已知弱点。
- **两者结合**产生大跳跃。多个 agent 之间的多轮交换驱动了收益。

### 为什么有效

两个机制：

1. **暴露于分歧。** 当一个 agent 看到另一个 agent 的推理链得出不同结论时，它必须要么辩护要么更新。无论哪种方式，第 r+1 轮的上下文都比第 r 轮更丰富。
2. **相关错误减少。** 在自一致性中，所有样本来自同一模型，所以错误相关——你平均得到一个自信的错误答案。不同模型或不同种子去相关。不同的*辩论观点*进一步去相关。

### 异构辩论

A-HMAD 和相关后续工作为不同 agent 使用*不同基础模型*。Llama + Claude + GPT 辩论减少了单一文化崩溃（Lesson 26），因为一个模型族的相关错误不被其他模型共享。

缺点：参与辩论的弱模型可能将共识拖向其错误答案（见"Should we be going MAD?"，arXiv:2311.17371）。

### NLSOM — 129 agent 扩展

Zhuge 等人（"Mindstorms in Natural Language-Based Societies of Mind"，arXiv:2305.17066）将这个想法扩展到 129 成员的社会。结果：专业化和自组织随规模涌现，系统在视觉问答等任务上超越单 agent。

### 失败模式

- **谄媚级联。** 所有 agent 顺从听起来最自信的那个 agent。辩论崩溃为最响亮的声音。为对抗性角色提示（"一个 agent 必须论证反面立场"）有帮助。
- **话题漂移。** 多轮辩论偏离原始问题。缓解：每轮重新注入问题。
- **计算爆炸。** N 个 agent × R 轮 = N·R 次 LLM 调用，每次上下文都在增长。5 agent、5 轮辩论是 25 次调用，上下文不断增长。每个问题的成本可能超过单次 CoT 调用的 10 倍。

## Build It

`code/main.py` 在一个数学问题上运行 3 agent × 3 轮辩论，每个 agent 从不同的（可能错误的）答案开始。Agent 是脚本化的——每个通过按脚本化置信度加权平均邻居的答案来"更新"。收敛在逐轮日志中可见。

演示展示两个关键效果：

- 单轮交换使 agent 更接近正确答案。
- 第 2 轮之后的额外轮次显示递减收益（与 Du 等人的饱和一致）。

运行：

```
python3 code/main.py
```

## Use It

`outputs/skill-debate-configurator.md` 为新任务配置辩论：agent 数量、轮数、异构性（相同模型 vs 混合）、角色分配（对称 vs 一个对抗性）。它还在运行前估计 token 成本。

## Ship It

如果你部署辩论：

- **轮数上限为 3。** Du 等人表明 3 轮捕获了大部分收益。更多是成本，不是质量。
- **Agent 上限为 5。** 超过 5，上下文膨胀和成本主导。
- **默认异构。** 池中至少两个不同的基础模型。
- **对抗性槽位。** 一个 agent 被提示无论如何都要反对。打破谄媚。
- **记录每一轮。** 隐藏中间轮次的辩论系统无法调试或审计。

## 练习

1. 运行 `code/main.py`，然后将轮数设为 5 并观察递减收益。在哪一轮额外收敛停止？
2. 添加第四个具有对抗性角色的 agent：始终反对当前多数。这是破坏还是改善了收敛？
3. 打印每轮的一致性分数（在多数答案上的 agent 比例）。它何时达到 1.0，这是否等同于"正确"？
4. 阅读 Du 等人第 4 节消融实验。使用此代码复制"仅 agent"vs"仅轮数"vs"两者"的结果。
5. 阅读"Should we be going MAD?"（arXiv:2311.17371）并列出轮询之外的两种辩论变体——例如裁判主导、辩论链、对抗性。

## 关键术语

| 术语 | 通俗说法 | 实际含义 |
|------|---------|---------|
| 心智社会 | "Minsky 的想法" | 智能作为交互专家；1986 框架现通过 LLM 辩论操作化。 |
| 多 agent 辩论 | "Agent 争论" | N 个 agent 提出、批评彼此、修订经过 R 轮，多数投票。 |
| 共识 | "他们同意了" | 不是认识论真理——只是多数答案上的比例。可能自信地错误。 |
| 轮次 | "交换步骤" | 一轮 = 每个 agent 读取其他人并更新一次。 |
| 异构辩论 | "混合模型族" | 使用不同基础模型来去相关错误。 |
| 谄媚级联 | "所有人同意最响的那个" | 辩论失败，agent 不顾正确性顺从最自信的 agent。 |
| NLSOM | "129 agent 社会" | 自然语言心智社会；Zhuge 等人的扩展版本。 |
| 相关错误 | "同模型，同 bug" | 为什么自一致性饱和；跨不同观点的辩论去相关。 |
