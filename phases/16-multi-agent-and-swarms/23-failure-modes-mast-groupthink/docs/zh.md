# 失败模式 — MAST、群体思维、单一文化、级联错误

> 2026 年的参考分类法是 **MAST**（Cemri et al., NeurIPS 2025, arXiv:2503.13657），源自 7 个最先进开源 MAS 的 1642 条执行追踪，显示 **41–86.7% 失败率**。三个根类别：**规格问题**（41.77%）——角色模糊、任务定义不清；**协调失败**（36.94%）——通信中断、状态不同步；**验证缺口**（21.30%）——缺少验证、没有质量检查。**群体思维**族（arXiv:2508.05687）补充：单一文化崩溃（相同基础模型 → 相关失败）、从众偏差（智能体强化彼此的错误）、心智理论缺陷、混合动机动态、级联可靠性失败。级联示例：重试风暴，支付失败触发订单重试，触发库存重试，压垮库存服务（几秒内 10 倍负载——需要熔断器）。记忆投毒：一个智能体的幻觉进入共享记忆，下游智能体将其视为事实；准确率逐渐衰减，使根因诊断痛苦。**STRATUS**（NeurIPS 2025）报告通过专业化的检测/诊断/验证智能体实现 1.5 倍缓解成功率改善。本课将失败模式视为一等工程目标。

**Type:** Learn
**Languages:** Python (stdlib)
**Prerequisites:** Phase 16 · 13 (Shared Memory), Phase 16 · 14 (Consensus and BFT), Phase 16 · 15 (Voting and Debate Topology)
**Time:** ~75 minutes

## 问题

多智能体系统在真实任务上 41-86.7% 的时间失败（Cemri et al. 2025 在 7 个开源 MAS 上测量了这一点）。这不能通过"加更多智能体"来调试。失败有结构性原因。MAST 分类法给你类别。本课将每个类别映射到具体的检测、诊断和缓解模式，使数字不再看起来任意。

2026 年的生产实践是将失败模式视为设计输入。你的架构不算"足够好"，直到你能指向每个 MAST 类别并说出你部署的缓解措施。

## 概念

### MAST 类别

**规格问题（41.77% 的失败）。** 智能体的任务定义不够紧密。示例：

- 角色模糊：两个智能体都认为自己是审查者。
- 任务欠规格："summarize this"但用户想要特定角度。
- 成功标准隐式：智能体无法判断自己是否成功。

缓解措施：
- 写显式角色合约。每个智能体的提示说明它做什么*以及不做什么*。
- 每个任务的验收测试。在智能体开始前，定义"完成看起来像 X"。
- 预检规格检查：一个单独的智能体在分派前审查任务定义。

**协调失败（36.94%）。** 通信或状态中断。

示例：
- 两个智能体在没有同步的情况下更新共享状态。
- 智能体间消息丢失（队列失败、超时）。
- 状态漂移：智能体 A 认为任务完成；智能体 B 仍在执行。

缓解措施：
- 带乐观并发的版本化共享状态。
- 关键消息的显式确认（重试直到确认）。
- 周期性状态同步检查点；尽早检测漂移。

**验证缺口（21.30%）。** 没有对输出的独立检查。

示例：
- 一个智能体声称成功；没人验证。
- 智能体链中每个都信任前一个的输出。
- 涌现组合行为缺少测试覆盖。

缓解措施：
- 独立验证器智能体（Lesson 13）。只读，独立源访问。
- 显式交接合约："A 的输出必须通过检查器 C 后 B 才开始。"
- 结果日志用于事后分析。

### 群体思维族（arXiv:2508.05687）

智能体同质化或互相模仿时的五种相关失败：

**单一文化崩溃。** 相同基础模型或训练数据 → 相关错误。当三个智能体共享一个 LLM 时，它们共享其幻觉。

**从众偏差。** 智能体向最响亮或最自信的同伴调整，即使对方是错的。

**心智理论缺陷。** 智能体无法建模彼此的信念；协调崩溃（Lesson 18）。

**混合动机动态。** 激励部分对齐的智能体漂向折中中间，不满足任何人。

**级联可靠性失败。** 一个组件的错误模式触发依赖组件的错误模式。

### 级联示例 — 重试风暴

一个经典的 2026 年事件模式：

```
payment service fails 10% of requests
   ↓
order agent retries payment (exponential backoff but naive)
   ↓
each retry is a new order-inventory check
   ↓
inventory service sees 2x normal load
   ↓
inventory service starts timing out
   ↓
every order retries inventory check
   ↓
inventory service sees 10x normal load
   ↓
cluster goes down
```

修复是经典的：**熔断器**。当下游错误率超过阈值时，用缓存或默认结果短路。加上每请求的有上限重试预算。

熔断器是你直接从分布式系统借用而无需修改的少数多智能体失败缓解之一。

### 记忆投毒（重访）

来自 Lesson 13：一个智能体的幻觉成为共享记忆事实；下游智能体基于被投毒的事实推理。在 MAST 术语中，这是共享记忆层的验证缺口。

逐渐的准确率衰减是症状。你不会得到崩溃；你得到难以追根溯源的缓慢漂移。

缓解：只追加日志、来源追踪、不可写验证器。已在 Lesson 13 中覆盖。

### STRATUS — 用于失败检测的专业化智能体

STRATUS（NeurIPS 2025）报告当你部署以下时缓解成功率改善 1.5 倍：

- **检测智能体。** 监视症状模式（高分歧、重试尖峰、准确率漂移）。
- **诊断智能体。** 给定症状，从 MAST 分类法推断可能的根因。
- **验证智能体。** 缓解措施应用后，检查症状是否消除。

这是 SRE 风格的事件响应，应用于智能体系统。三个角色都可以是带专业化提示的 LLM 智能体。

### 失败模式审计

2026 年的最佳实践是年度（或每次重大发布）失败模式审计：

1. **追踪采样。** 收集 ~1000 条真实执行追踪。
2. **分类。** 对每条追踪的失败，映射到 MAST + 群体思维类别。
3. **计算按类别的失败率。** 哪些类别主导你的系统？
4. **排名缓解措施。** 哪个修复能消除最多失败？
5. **选择 2-3 个缓解措施。** 实施；下季度重新审计。

纪律比具体选择更重要。没有审计，失败混入噪声中永远不会被系统性地解决。

### 当系统静默失败

最危险的失败类别是静默正确性失败。一个大声失败的系统（崩溃、异常、告警）可以被监控。一个产生看似合理但错误输出的系统无法通过异常日志检测。这就是为什么验证缺口是每次失败最昂贵的类别，尽管按计数只有 21.30%。

投资于：
- 基于采样的人工审查。
- 黄金数据集回归测试。
- 重要输出的跨智能体交叉检查。

### 失败 vs 慢失败

有些失败是即时的；有些是缓慢的。即时失败（超时、schema 不匹配、认证错误）检测成本低。慢失败（记忆投毒、单一文化漂移、角色模糊）检测和预防成本高。

2026 年的工程手段：检测慢失败代理指标，以便在漂移变成可见错误之前捕获它。一致率、重试率、输出长度分布和连续智能体版本之间的编辑距离都是有用的代理指标。

## 上线清单

生产中的失败模式纪律：

- **每季度 MAST 审计。** 不是年度。类别随系统增长而变化。
- **到处都是熔断器。** 每个对任何依赖服务的出站调用。默认打开阈值在 5-10% 错误率。
- **黄金数据集。** 小型、高质量、人工审计。每周对其进行回归测试。
- **STRATUS 三人组。** 检测 + 诊断 + 验证智能体监控生产。先只用检测智能体；当症状嘈杂时添加诊断。
- **失败预算。** 按类别的显式 SLO 失败率。超出预算触发停止发布对话。

## 练习

1. 运行 `code/main.py`。确认熔断器封顶了重试风暴。变化失败阈值并观察权衡。
2. 实现一个**慢失败代理指标**：3 个并行智能体的一致率。当它急剧下降时触发告警。通过逐渐关联智能体输出模拟单一文化漂移。
3. 阅读 Cemri et al.（arXiv:2503.13657）。选择他们 7 个 MAS 系统中的一个并映射其前 3 个失败类别。这些与 MAST 预测的相比如何？
4. 阅读群体思维论文（arXiv:2508.05687）。识别五种模式中哪种在生产中最难检测。提出一个代理指标。
5. 为你了解的特定多智能体系统设计 STRATUS 风格的检测-诊断-验证三人组。检测监视哪些症状？诊断推荐什么缓解措施？验证如何确认它们有效？

## 关键术语

| 术语 | 通俗说法 | 实际含义 |
|------|---------|---------|
| MAST | "2026 分类法" | Cemri 2025；3 个根类别 + 14 个子类型的失败。 |
| Specification Problem | "角色模糊" | 任务或角色定义不足；智能体不知道该做什么。 |
| Coordination Failure | "状态漂移" | 智能体间的通信或同步中断。 |
| Verification Gap | "没人检查" | 输出被接受而没有独立验证。 |
| Groupthink family | "同质性失败" | 单一文化、从众、心智理论缺陷、混合动机、级联。 |
| Monoculture collapse | "相同模型，相同幻觉" | 来自共享基础模型或训练数据的相关错误。 |
| Retry storm | "级联错误放大" | 一个失败触发重试，重试放大下游负载。 |
| Circuit breaker | "错误率快速失败" | 当错误率超过阈值时打开；用默认值短路。 |
| STRATUS | "事件响应三人组" | 检测 + 诊断 + 验证智能体。1.5 倍缓解成功率。 |
| Memory poisoning | "幻觉传播" | 共享记忆事实被污染；下游智能体基于毒物推理。 |
